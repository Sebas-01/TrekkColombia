Modelo Entidad-Relación

Estructura lógica de la Base de Datos de Trekking App

erDiagram USUARIO ||--o{ FAVORITOS : "marca_como" RUTA ||--o{ FAVORITOS : "es_marcada_por" USUARIO { int idusuario PK string nombre string correo string password string telefono string rol } RUTA { int id PK string title string difficulty string duration geometry geom } FAVORITOS { int idusuario FK int idruta FK }

Usuarios

  • Almacena exploradores y admins.
  • Clave principal: idusuario.
  • Roles: 'usuario' y 'admin'.

Rutas

  • Datos técnicos de los senderos.
  • Geolocalización avanzada (PostGIS).
  • Incluye imágenes y guías.

Favoritos

  • Tabla de enlace (N:M).
  • Conecta a usuarios con sus rutas.
  • Permite la personalización del feed.
